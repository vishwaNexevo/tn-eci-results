# Tamil Nadu Assembly Election Results 2026 — Live Dashboard

Production-ready **Next.js 14 (App Router) + TypeScript + Tailwind CSS** dashboard
for the live ECI results JSON.

## Highlights

- **SSR-first SEO**: Live data is fetched on the server in `app/page.tsx` and rendered into the HTML so crawlers see real numbers.
- **Rich metadata**: Dynamic `<title>` / `<meta description>` based on the current leading party, Open Graph + Twitter cards, canonical URL, geo tags, robots directives.
- **JSON-LD structured data**: `Event`, `Dataset`, `BreadcrumbList`, and `ItemList` schemas embedded as `<script type="application/ld+json">`.
- **`sitemap.xml` + `robots.txt`** generated by `app/sitemap.ts` and `app/robots.ts`.
- **Live polling** every 20s via a custom `useLiveResults` hook — no full page reload, with cleanup, in-flight refresh state, and a flash animation when a constituency flips party.
- **API proxy** at `/api/results` so the upstream JSON can be served from the same origin (CORS-safe).
- **Light theme only**, mobile + desktop responsive, party colors taken from the API with sensible fallbacks for DMK / AIADMK / BJP / INC / TVK / NTK / PMK / DMDK / CPI / CPI(M) / VCK / MDMK / IUML / IND / Others.
- **Filtering, search, and sorting** preserved during polling.
- **No hardcoded results** — every number comes from the API.

## Components

- `ResultHeader`, `LiveUpdateIndicator`
- `SummaryCards`
- `PartyCard`, `PartyChart`, `Comparison`
- `FilterBar`, `ConstituencyTable`, `PartyBadge`
- `Dashboard` (composes everything, manages state)

## Run locally

```bash
npm install     # or pnpm install / bun install
npm run dev     # http://localhost:3000
```

## Configure canonical URL

Set `NEXT_PUBLIC_SITE_URL` in `.env.local` so canonical/OG/sitemap URLs are correct:

```
NEXT_PUBLIC_SITE_URL=https://your-domain.com
```

## Data source

`https://results.eci.gov.in/ResultAcGenMay2026/election-json-S22-live.json` (S22 = Tamil Nadu)

The endpoint exposes for each of the 234 constituencies:
`[partyCode, stateCode, seatNumber, candidateName, color]`.
The dashboard does **not** render fields the API does not provide (vote counts, margins, constituency names) — so nothing is fabricated.
